Web

题目来源：XCTF Web 进阶区 Cat

​ 根据题目要求，要求输入域名，我们试着输入127.0.0.1

接着试着用管道执行命令，输入127.0.0.1 | ls，返回Invalid URL

因为该题目是通过url参数传递数据，所以应该可以传递url编码

试着%70是没问题可以正常传输的，此时显示的是 url=p，如图：

但当我们试着采用url=%80时候，出现了报错

从报错信息可以发现，后台至少运行了php和django，如图：

经了解，

全部数据使用HTTP协议中的 “POST” 操作来发送。 要发送文件，在文件名前面加上@前缀并使用完整路径。 文件类型可在文件名后以 ‘;type=mimetype’ 的格式指定。 这个参数可以是 urlencoded 后的字符串，类似’para1=val1&para2=val2&…’，也可以使用一个以字段名为键值，字段数据为值的数组。 如果value是一个数组，Content-Type头将会被设置成multipart/form-data。 从 PHP 5.2.0 开始，使用 @ 前缀传递文件时，value 必须是个数组。 从 PHP 5.5.0 开始, @ 前缀已被废弃，文件可通过 CURLFile 发送。 设置 CURLOPT_SAFE_UPLOAD 为 TRUE 可禁用 @ 前缀发送文件，以增加安全性。

那么此时我们就可以看看django的配置文件，通过url搭配@来查看，后缀为：

?url=@/opt/api/api/settings.py

此时可以打开这个settings文件，我们直接CTRL+F来查看database，如图：

可以看到，数据库为sqlite3，那么我们直接再来看看它的数据库，同样，通过url和@来查看

?url=@/opt/api/database.sqlite3

同样，直接CTRL+F来看看有没有CTF的 信息

因此，拿到了falg：WHCTF{yoooo_Such_A_G00D_@}
